The bug that lived for twenty-seven years

An AI model just found thousands of zero-day vulnerabilities in the software the world runs on. Its creators decided not to release it. Here is why that decision matters more than the model itself.

Last month, an AI model built by Anthropic was set to work reading the source code of OpenBSD, the operating system used inside firewalls and servers around the world thanks to its reputation for being unusually well-audited. Within a few hours, the model found a flaw in the networking code that had been sitting there since 1998. Anyone who knew about it could have used it to remotely crash any machine running the software. Twenty-seven years, and no human auditor or automated tool had spotted it.

The model is called Claude Mythos Preview, and the OpenBSD bug is one of thousands of similar previously-unknown vulnerabilities (called zero-days) it has now found across every major operating system and web browser. Among them: a sixteen-year-old flaw in FFmpeg, the video library running inside almost every streaming service; exploitable chains in the Linux kernel that hand an attacker root access; and a working browser exploit that stitched four separate bugs together to escape a security sandbox.

Zero-day

A security flaw nobody yet knows about — not the developers, not the users, not the security community. The most dangerous class of software bug, because they can be used before anyone knows to defend against them. Nation-states stockpile them. Criminal groups pay millions for them.

On 7 April, Anthropic announced this work, and announced at the same time that they would not be releasing the model to the public. The reason for withholding it is the same reason it is interesting: whatever lets you find a vulnerability lets you exploit one. Mythos Preview now writes working exploits for over 72% of the Firefox JavaScript engine flaws it discovers, up from almost zero in the previous-generation model only weeks ago. Released openly, it would be the most powerful tool ever handed to whoever bothered to download it.

Instead, Anthropic has set up a programme called Project Glasswing, which gives access to a controlled group: twelve large technology firms (Microsoft, Google, Apple, AWS, CrowdStrike, NVIDIA, Palo Alto Networks among them) and around forty other organisations that maintain critical infrastructure. Crucially, that group includes the Linux Foundation, the Apache Software Foundation, and the Open Source Security Foundation. Anthropic has put USD $100M in usage credits behind the programme and given a further USD $4M directly to those open-source security bodies, so the volunteer maintainers of the software the world quietly depends on are not locked out of the same tools the trillion-dollar firms are using.

That distinction matters more than it sounds. A great deal of the internet's plumbing — Linux itself, OpenSSL, FFmpeg, Apache web servers — is maintained by small numbers of underpaid or unpaid people. If only the giants get the AI tools to find vulnerabilities, the giants get safer software while everyone else's stack rots. Glasswing is, at least in part, an attempt to avoid that outcome.

The argument

Withholding a capability like this is a choice. It is not a property of the technology, and it is not the default behaviour of a frontier AI lab.

Anthropic decided to do it. Other labs working at the same level (OpenAI, Google DeepMind, Meta, the leading Chinese labs) are pursuing the same problems on roughly the same timescales. They will reach the same capability soon. The open question is what they will do with it when they do.

The track record so far gives some sense of who tends to do what.

OpenAI
Image generation: shipped to public the day of announcement. 130M users, 700M images in week one. GPT-5: reached free-tier users within 24 hours of launch.
Anthropic
Computer Use: kept in beta for over a year before general release. Mythos Preview: not being released at all.

These are very different institutional habits around a very similar set of choices.

A separate problem is open weights. Meta's Llama models and DeepSeek's R1 are released as downloadable files: anyone can run them, modify them, fine-tune them, and once they are out, no one can take them back. A Cisco study found DeepSeek R1 failed to refuse even one of fifty harmful prompts tested against it. A US government evaluation in September last year found it would comply with 94% of overtly malicious requests. If a Mythos-class model is released as open weights at any point in the next few years, the question of restraint stops being meaningful, because there will be nothing left to restrain.

The software supply chain was not built for this kind of throughput, and is being asked to absorb roughly a year's worth of vulnerability disclosures in a few months.

· · ·

So expect, over the next while, an unusually heavy run of software updates. The reason for them is worth knowing. Underneath the patches is a more uncomfortable story: that a great deal of our current safety depends on which company happened to build the dangerous thing first, and what they happened to decide to do with it.
